STEP 1: Get some background...

What is HTML? Why Learn HTML? What are the standards? What are the components of a web page?

What is HTML?

HTML stands for Hyper Text Markup Language. It is used to structure web pages by using using "tags". HTML is a computer language , but it is not cosidered a programming language. This can be confusing to understand or to explain, especially if you do have a basic knowledge of the different types of computer languages (Read more about it here*). Nevertheless, this does not mean HTML does not require any skills. It does. You need to plan how to structure the information you are presenting. You also need to know what elements to use and what are the current standards for making your page usable and accessible. There can be a lot more going on an HTML page than one could think!

Why learn HTML?

Learning HTML is the basic and the most essential step for starting with web development and/or web design (More information on "Web Developer vs. Web Designer" here* and here*). And... I do not think I need to state the importance of the web and the internet in today's world. So, let me just say technology is relevant today and it will still be relevant tomorrow. This includes the internet and the web, of course. Some could argue you could do fine without HTML and still be able to design and develop a web page. This is somewhat true, for some people and some simplistic purposes, but I do not think it is true for a web designer or a web developer. Read more about it here*.

What are the standards?

There are many ways to make a page with HTML. Over time, there have been approaches to have some standards for the proper use of the elements in HTML. The most current version of HTML is HTML5. HTML5 has been updated to focus on using the proper elements for a specific purpose. Creating a web page involves much more than simply having a "pretty" presentation of your information. You need to think about usability and accesability. Yes, users and clients want a page that looks good, but the page also needs to be easy to use for everyone. This, then, remarks the importance of both usability and accesability. It might be even more time-consuming to create a page including these three components (aesthetics, usability, and accesability), but it has become a need to do so. John Rempel talks about the importance of accesability in this video from AccessibleMediaTube on YouTube. The W3C is an international community developing web standards. Go to the www.w3.org to check current recommended standards. They even have a "Markup Validation Service," where you vcan alidate your page follows standards (validator.w3.org). This validator is a good tool, but it certainly does not check every single mistake. An easy way to remember how you should build your page is by separating aesthetics from functionality and by using elements that give the intended meaning to what you are presenting.

What are the components of a web page?

Like previously mentioned, tags help structure your page. But, you need to think about which tags you should use to give meaning to that specific block of information. Structure also means that you need to plan where will you locate the information you are presenting. And, how do you plan a good structure for your web page? Well, you take functionality into consideration. What is the main fuction for your page? To publish a portfolio, to write a blog, to teach an online class, to share an article, to promote your business... There are many reasons someone would like to make a web page and there are many ways to give it structure. It all depends how you want the user to navigate through your page, and how you want the user to actually utilize your page. For example, you could have a navigation panel on top of your page or you could decide to add on the left. The important thing is to take time to plan what you will do. Also, do not forget about content! The structure of your page will not matter if you do not have content. But, content not only includes text. You could have images and other media (like videos, audio, PDFs, etc..). Content could also help you add a better look and feel to your page, but you must remember that everything you include in your page should enhance your content without making distractions! So, adding flashy images, for example, does not always make your page better. Ultimately, most web pages today are interactive, so it is a good idea to add ways to make the use of your page an interactive experience. Making a web page with HTML, though, is most of the times not enough for adding interactivity to your page. Remember that HTML's purpose is adding structure to your pages. HTML is a markup language, so you will need to learn script languages like Javascript and PHP to add more fuctionality and interactivity to your page (This is the Wikipedia link to "Scripting language"). Moreover, it has already been mentioned that aesthetics is an important aspect to consider when making web pages, but can you make a page look "pretty" using only HTML? You certainly can, to a certain extent. You actually are currently looking at an example of a page using only HTML. Yes, I know, this looks simple. There are ways to add colors and background images in HTML, as well, but that usually is not recommended. You do not want to mix content with style, so learning CSS (Cascading Style Sheets) is also important. Styling with CSS and linking it to your page is an usual way of doing this. You will see the relevance of this concept when you get to make a web page with HTML and decide to try different style templates. You certainly do not want to be creating different HTML pages, only different styles. So, you will simply use tags. ids, and classes and you will separately define the style fo them. For now, we only worry about HTML. You need to understand HTML before doing CSS, and especially before you try programming with script languages.

STEP 2: Learn about the basic HTML components!

What are tags? How are tags displayed? What are some tag attributes? Are all symbols correctly displayed? What tags can I use for semantic purposes?

What are tags?

So, we know HTML is composed of tags. You use tags to define blocks of content. But, what are tags and how do you use them? Tags start and end with a bracket (e.g. <html>). All tags have a beginning and an end, but not all do it the same way. There are:

  • tags that define beginning and end with a start tag and an end tag (e.g. <body> </body>), and
  • self-closing tags (e. g. <img src="img.jpg"/>)

Some tags have attributes, like:

  • src
  • href
  • target

Attributes are always specified in the start tag!

How are tags displayed?

There are two most common types of display for elements: block and inline. The following shows some examples and descriptions for each:

  • block: a new line is inserted before and after the element and it can take witdh and height
    • Headings
      • <h1>, <h2>, <h3>... <h6>
      • They have syntax and semantics.
      • Should not be used solely for their appearance
      • The closing tag should be added! (e.g. <\h1>)
    • Paragraphs
      • <p> </p>
      • Only contain inline elements
    • Div
      • <div> </div>
      • Generic section
    • Lists
      • Ordered
        • <ol> </ol> (to start and end)
        • <li> </li> (to list each item)
        • Usually presented in bullets, but not always.
      • Unordered
        • <ul> </ul> (to start and end)
        • <li> </li> (to list each item)
        • Usually they are numbered (1,2,3...), but you can change what numbering to use (I, II, III...).
    • Line Break
      • <br/>
      • Adds a visible line break to the web page
  • inline: uses only the needed space for the element and does not take width and height
    • Images
      • <img src="img.jpg" alt="logo"/>
      • Needs attributes to display an image:
        • src:
          • source link for the image
          • It's required to show an image!
        • alt:
          • tells what an image is.
          • Not really required to show an image, but needed to follow HTML5 standards
          • Useful if the picture does not show
          • Helpful for search engine optimization
          • Needed to add more accesability and those who use screen readers
        • title:
          • Displays on hover
        • class:
          • Commonly used for extra formatting (with CSS)
          • This attribute can be used in all other tags!

Remember that tags can have syntax and semantics. In other words, there are different ways to do something, but not all ways are recommendable because you should use the tags that were designed to better describe what you are doing. For example a <div> could be use to enclose some group of information, but it does not really tell what it is (since it is a generic tag). HTML5 has more tags with semantics that you could use to describe such block of content.

What are some tag attributes?

There are special attributes that can apply to any tag, and the most used are:

Are all symbols correctly displayed?

As you know, tags are not displayed in the browser. They exist to give structure and define meaning of the content. For that reason, and to avoid confusion, there are special entities to display some special symbols like the following:

  • the starting bracket ( < )
  • the ending bracket ( > )
  • the copy sysmbol ( © )
  • blank space (   )
  • cent symbol ( ¢ )
  • ampersand ( & )

You can look at this page's source code to check that they indeed display what they are supposed to display!

What tags can I use for semantic purposes?

You know you should design you web page before you start typing any HTML tags and that you should use semantics as much as possible. But, what are some tags that could help define content to have a better, easy-to-use, accessible web page? The following are some semantic tags used in HTML5 that follow standards:

  • <header> </header>
    • Not to be confused with the <head> tag
    • This tags encloses the tags and information to be included in the header of your page
    • It does not do anything to the display, but it describes what you have in your page for accessibility puposes
  • <nav> </nav>
    • Section of the page that links to other pages
    • Your navigation panel is added here
    • You could add this to the header
  • <footer> </footer>
    • Information usually found at the bottom
    • It could include copyright information, company address...
    • It is not required and it will not locate your information at the bottom but gives semantic to your code.
  • <figure> </figure>
    • It gives more semantics than just adding the img tag
    • This is where you encapsulate an image and the information for that image.
    • To add information you can use the figcaption tag inside the figure tag and after the img tag
    • Useful for accessibility since it could tell more information about an image to a person who can't see it

But, there many more used for different purposes. Some examples are:

  • Structural Elements
    • <article> </article>
    • <aside> </aside>
    • <main> </main>
    • <menuitem> </menuitem>
    • <summary> </summary>
    • <section> </section>
  • Form Elements
    • <datalist> </datalist>
    • <keygen> </keygen>
    • <output> </output>
  • Input Types (Attributes)
    • color
    • date
    • email
    • list
  • Graphic Elements
    • <canvas> </canvas>
    • <svg> </svg>
  • Media Elements
    • <audio> </audio>
    • <embed> </embed>
    • <source> </source>
    • <track> </track>
    • <video> </video>

STEP 3: Practice, practice, practice!

IMAGES LINKS VIDEO AND AUDIO TABLES

There is no better way to learn HTML, or any computer language, thant to start practicing and creating new things with it! Before you do, read how you can add images, links, video and audio, and tables below.

IMAGES

Images are an important part of most web pages. Remember you need the img tag to add an image to you page and enclosing all the image information inside the figure tag. In the source attribute, you should include the image location and its file extension (.JPG, .GIF, .PNG). So, if you have image in the same folder as you web page, you could simple type the name of the image and its extension type (e.g. "my-image.jpg" ). If it is inside a folder, add that location (e.g. "my-img-folder/my-image.jpg"). You can also add an image from a different source not found in your directory (e.g. "https://upload.wikimedia.org/wikipedia/commons/thumb/7/79/Face-smile.svg/2000px-Face-smile.svg.png).

You can add a favicon to your page. It is located next to the title and it must be included in the head. The following is the code for of adding a logo as your favicon:

< link rel="icon" type="image/png" href="icon/my-logo.png" / >

You cannot use jpegs or gifs, only a special icon image or a .png!

Always add the alt atttribute to all your images. It is important for accessibility issues. If you do not have anything important to mention about the image, you can leave it blank, but do not omit it. Remember that the alt attribute serves for accessibility issues.

LINKS

Links are known as "anchor links" and they make up the web. Most all pages have links. You can add a link to you page with the <a> tag, which stands for "anchor link". The following is the code so you add a link to your page:

<a href="http://wwww.utrgv.edu">The University of Texas Rio Grande Valley</a>

This is the actual link to UTRGV!

An anchor link requires the href attribute so you can tell the location to link to and content so you can click the link. In the example above, "http://www/utrgv.edu" is the location the new content is located and the text "The University of Texas Rio Grande Valley" is the clickable content. There are different types of links, the most common are listed below:

  • Absolute
    • The above example is an absolute reference
    • Someone else's page
  • Relative
    • Links to a local file in the same folder, in a different folder, or inside the same document.
      • Same folder: href="page2.html"
      • Other folder: href="examples/page2.html"
      • Same document: href="#references"
  • Graphical
    • You can add clickable images that take you to a link!
    • Just add the image in the content part between the beginning and end tags.
    • Do not forget the alt attribute!

You can add a target to an anchor link with the target atttribute (e.g. _self, _blank )

VIDEO AND AUDIO

There is a video tag and an audio tag, which was not available in older versions of HTML. As the name says it, the video tags is used to add video to your page. If you add text inside the video tags, the text will appear if a browser does not support this tag. Since this might not be supoorted in all browsers, you should provide links to any needed plugins. The following are attributes to better present your video or audio:

  • height and width
  • autoplay
  • controls
  • loop

TABLES

Tables are very common but they have been missused in the past. They are not supposed to be used for layout. Their purpose is to display tabular data. Here are some steps to make a table:

  1. Plan and design the table you want to display.
  2. Start with the <table> tag
  3. Then, add a <tr> tag to denote a row.
  4. Add the <td> tag to add a cell in the row you are at.
  5. Add the content you wish to display in that cell
  6. Close with the </td> tag.
  7. Repeat steps 4-6 for all the cells in the row you are at
  8. Then, clode <tr> to end that row.
  9. Repeat steps 2-8 for each row.
  10. Close with the </table> tag when you finish with all the rows.

You can also add headings in the first row. Simply add the <th> tags instead of the >td> tags.
